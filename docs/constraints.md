# 制約仕様（Constraints）

## 目的

研究・実験リポジトリにおける品質・コスト・安全性の制約を定義する。

## 制約の適用範囲

- すべての処理は、実行前に制約を評価する。
- 判断不能な場合は安全側に倒す（フェイルクローズ）。

## 制約一覧

### C-001 実験コード品質基準

| パラメータ | 値 | 備考 |
|---|---|---|
| 適用範囲 | `experiments/` | 実験コードに適用 |

- 判定：ruff check は必須、mypy / テストは推奨
- 理由コード：`C001_experiment_quality`

### C-002 昇格品質基準

| パラメータ | 値 | 備考 |
|---|---|---|
| 適用範囲 | `src/` | 昇格後のコードに適用 |

- 判定：mypy strict / テスト必須 / ruff check 必須 / docstring 必須
- 理由コード：`C002_promotion_quality`

### C-003 フィードバック前提条件

| パラメータ | 値 | 備考 |
|---|---|---|
| 適用範囲 | dev-orchestration-template へのフィードバック | フィードバック時に適用 |

- 判定：ベンチマーク結果が存在し、有効性が確認されていること
- 理由コード：`C003_feedback_precondition`
