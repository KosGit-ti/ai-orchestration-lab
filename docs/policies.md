# ポリシー（Policies）

## 最重要ポリシー（違反したら即中止）

### P-001 禁止操作

- 外部 API への意図しない接続を禁止する（研究目的で明示的に許可されたものを除く）。
- 研究で外部 API を使用する場合は ADR に記録し、`.env` 経由で認証情報を管理する。
- CI の `policy_check.py` で機械的に検査する。

### P-002 秘密情報禁止

- API キー、トークン、認証情報、個人情報をリポジトリに含めない。
- `.env` はローカルのみとし、`.env.example` は変数名のみを記載する。
- CI の `policy_check.py` で秘密情報パターンを検査する。

### P-003 制約最優先

- 制約回避（スキップ、無効化、ハードコード回避）は禁止する。
- 例外が必要な場合は ADR を作成し、期限と復帰条件を定義する。

## 安全停止

### P-010 フェイルクローズ

- 判断不能（データ不整合、欠損、未定義状態）の場合は安全側に倒す。
- 疑わしい場合は処理を中止する。

### P-011 異常時停止

- エラー連鎖、制約違反の連鎖時は安全に停止する。

## 監査と証跡

### P-020 監査ログ

- 重要な判断、制約判定、例外を記録する。
- ログには秘密情報を含めない。

### P-021 再現性証跡

- 実験の設定、シード、コミット SHA を記録し、再実行可能にする。

## 変更管理（正本運用）

### P-030 正本優先

- 仕様変更は `docs/requirements.md` または `docs/policies.md` に反映する。
- 計画変更は `docs/plan.md` を上書き更新する。
- 重要判断は `docs/adr/` に残す。

### P-031 変更粒度

- 1 PR で理解できる粒度に分割する。
- 複数の研究テーマを1つの PR に混在させない。

## 研究固有ポリシー

### P-040 実験コード分離

- 実験コードは `experiments/` に配置する。`src/` に直接配置しない。
- 実験コードが検証を経て安定したら `src/` に昇格する。

### P-041 依存管理

- 依存追加時はライセンスを確認する。
- 許可ライセンス：MIT / Apache 2.0 / BSD / ISC / PSF License
- 禁止ライセンス：GPL / AGPL / SSPL

### P-042 研究文書化

- 各研究テーマの調査結果、設計判断、検証結果は `docs/research/` に文書化する。
- dev-orchestration-template へのフィードバック方針を明記する。
